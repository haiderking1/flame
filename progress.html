What was built

Per-turn self-improvement loop landed

  • NudgeTracker (nudge-tracker.ts) — memory gate per user turn, skill gate per tool-call iteration; resets on fire and whenever skill_manage is used. 0 disables a gate.
  • runBackgroundReview (background-review.ts) — forks a headless Agent after qualifying turns, off the user-facing path. Reuses the parent streamFn/model/sessionId/base system prompt verbatim (provider prefix-cache parity), restricts tools to a memory + skills whitelist, shares the parent MemoryStore so writes hit disk, caps iterations, dedupes the action summary, best-effort.
  • Review prompts (review-prompts.ts) — the memory / skill / combined prompts ported near-verbatim from background_review.py, retargeted to flame tool names.
  • AgentSession wiring — tool-iteration counting in afterToolCall, memory gate at prompt start, fire-and-forget review after the run drains, emits a self_improvement_review event.

Inactivity curator core landed

  • curator.ts — deterministic age-based transitions (active → stale → archived) using each skill dir's most recent file mtime; pinned skills bypass; archive-never-delete (moves to .archive/); optional LLM consolidation fork using a ported curator prompt.
  • curator-state.ts — .curator_state (lastRunAt, pause, pins, per-skill lifecycle), atomic writes.
  • curator-backup.ts — pre-run recursive-copy snapshots + manifest, pruned to keep N.
  • Session-start trigger — gated by interval; emits a curator_run event. Opt-in via settings.

Roadmap to 100% hermes parity

1. Surface review & curator results in the TUI — effort: S

The self_improvement_review and curator_run events already fire. Render the hermes 💾 Self-improvement review: … line in the footer / notification area. Closes surfacing to ~95%.

2. /curator slash-commands — effort: M

Wire run (with --dry-run), pin/unpin, pause/resume, restore <name>, and status. State + pins + snapshot restore primitives already exist; this is mostly command plumbing. Closes most of the curator CLI gap.

3. Curator dry-run mode + per-run reports — effort: M

Port the dry-run banner (report-only, no mutations) and write REPORT.md + run.json per run under a logs dir, including the consolidations/prunings split. Hermes uses these for review-before-live.

4. Skill provenance + usage tracking — effort: L

Port a flame analog of hermes' skill_usage: created-at / last-activity / use-count + an agent-created vs imported distinction. Today the curator derives activity from file mtime and treats all skills as agent-created. This unlocks accurate transitions and the consolidation YAML reconciliation.

5. consolidations/prunings reconciliation — effort: M

Parse the consolidation pass's structured YAML block and classify each archived skill as consolidated-into-umbrella vs pruned by scanning that run's skill_manage calls (hermes' _classify_removed_skills). Drives skill-reference migration.

6. Iteration-accurate skill cadence — effort: S

Optionally switch the skill nudge to count one increment per assistant turn that issued tool calls (hermes' per-API-iteration semantics) instead of per tool call, for exact cadence matching.

7. tar.gz snapshots + rollback CLI — effort: S

Swap recursive-copy snapshots for streamed tar.gz (hermes parity, smaller on disk) and add a restore path that moves the current tree aside before extracting, so rollback is itself undoable.

8. Inactivity-accurate trigger — effort: S

Hermes runs the curator when the agent is genuinely idle (min-idle-hours since last activity), not just at session start. Add an idle gate so it never competes with an active turn.

Explicitly out of scope (by earlier project decisions)

  • No bundled skills (agent grows its own library — the stated source of value).
  • External memory providers (honcho / mem0 / supermemory), <memory-context> scrubber, session_search.
  • codex_app_server runtime downgrade for the review fork (flame has no codex runtime).
